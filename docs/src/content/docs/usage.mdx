There are two ways to display LilyPond scores when using `astro-lilypond`:

1. Write LilyPond code inside of fenced code block inside of a [Markdown](#markdown) file.
2. Import the [`<LilyPond>` component](#component) and pass it a `.ly` file.

## Markdown

Use a fenced code block with the `lilypond`, `ly`, or `ily` language tag:

<UsageExample />

The markdown-based approach is great for small musical examples and excerpts which you want to write alongside prose. You can write your text, compose some music, and then switch back to text without leaving the file!

### Cropping

All markdown blocks are cropped unless you set [`defaults.crop`](/configuration/#defaultscrop) to `false` in your config. Cropping cuts away margins and condenses all music into a single, sometimes large, image, and eliminates most spacing between staves.

### Alt text

Rendered scores get an `alt` attribute so they're accessible to screen readers.

By default, alt text is derived automatically from the `title` and `composer` fields in the score's `\header` block:

<FencedLilypond code={headerExample} />

This produces `alt="Sonata No. 14, by Ludwig van Beethoven"`. If only one of `title`/`composer` is present, alt text falls back to just the title, or `"Sheet music by {composer}"`. With neither, alt text is left empty (which marks the image as decorative, excluding it from screen readers).

A multi-page score's pages are wrapped in an `<ol>`, so every page gets the same alt text — screen readers already announce list position (e.g. "1 of 2") on their own.

You can override the automatically-derived alt text by setting `alt="..."` in the fenced code block's meta string, right after the language tag:

````md
```lilypond alt="Sonata in C# minor"
\header {
  title = "Sonata No. 14"
}
...
```
````

## Component

When you have longer, complex scores, or if you just prefer to keep music files separate from Markdown, you can use the Astro `<LilyPond>` component.

The `<LilyPond>` component renders notation in any `.astro` file. Import a `.ly` or `.ily` file and pass it to the `content` prop.

<Code code={BachInventionExampleRaw} lang="astro" title="MyComponent.astro" />

<BachInventionExample />

When a score renders to more than one page, as above, the pages are wrapped in an unstyled `<ol>` element which contains one `<li>` list item for each page `<img>`. You can style pages and containers however you want. See [styling](/styling) for more examples.

### Cropping

`<LilyPond>` imports follow [`defaults.crop`](/configuration/#defaultscrop) too, but the default value, `"markdown-only"`, only crops Markdown — component output is left as full, potentially multi-page, pages by default, preserving page margins and spacing.

Individual imports can override `defaults.crop` by appending `?crop` or `?nocrop` to the import path.

Append `?crop` to always crop the score:

```astro "?crop" title="CroppedScore.astro"
---
import LilyPond from "astro-lilypond/component";
import bachInvention from "./bach-invention.ly?crop";
---

<LilyPond content={bachInvention} />
```

Append `?nocrop` to force full, uncropped pages:

```astro "?nocrop" title="UncroppedScore.astro"
---
import LilyPond from "astro-lilypond/component";
import bachInvention from "./bach-invention.ly?nocrop";
---

<LilyPond content={bachInvention} />
```

### Alt text

`<LilyPond>` imports derive alt text automatically the same way [Markdown](#alt-text) does, from the imported `.ly` file's `\header` block. Pass an `alt` prop to override it with your own description:

```astro title="MyComponent.astro"
---
import LilyPond from "astro-lilypond/component";
import bachInvention from "./bach-invention.ly";
---

<LilyPond content={bachInvention} alt="Bach's Invention No. 1 in C Major" />
```

### Props

#### `content`

**Type:** `LilypondContent` (`{ srcs: string[] }`)  
**Required:** yes

The result of an imported `.ly` or `.ily` file. The output is resolved at build time according to the `defaults` configured in your integration.

#### `class`

**Type:** `string`

Additional class name(s) merged onto the rendered `<img>` element alongside the default `lilypond` class. See [Styling](/styling/) for how to use this for per-instance styles.

#### `style`

**Type:** `string`

Inline styles applied directly to the rendered element.

#### `alt`

**Type:** `string`

Overrides the automatically-derived alt text (see [Alt text](#alt-text)). Takes precedence over any `title`/`composer` found in the `.ly` file's `\header` block.
