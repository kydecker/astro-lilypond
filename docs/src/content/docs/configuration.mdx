Pass options to the integration when registering it in `astro.config.mjs`:

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import lilypond from 'astro-lilypond';

export default defineConfig({
  integrations: [
    lilypond({
      format: "svg",
      defaults: {
        version: "2.26.0",
        resolution: 144,
        crop: false,
        staffSize: 20,
        paperSize: "a4"
      },
      outputDir: "_lilypond"
    }),
  ],
});
```

Every option below is optional â€” the values shown are the defaults `astro-lilypond` uses when it's omitted.

## Options

### `defaults`

**Type:** `object`  
**Default:** `undefined`

Values applied when rendering each score. Any individual `.ly` file can still override them with its own settings.

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      format: "png",
      defaults: {
        version: "2.24.0",
        resolution: 300,
        crop: true,
        staffSize: 16,
        paperSize: "letter"
      }
    }),
  ],
});
```

#### `defaults.version`

**Type:** `string`  
**Default:** `"2.26.0"`

To know how to output text, LilyPond needs to know a `version` to use for compilation. This is usually defined at the start of each `.ly` file:

<FencedLilypond code={versionExample} />

By default, `astro-lilypond` will prepend this version for you, so you don't need to write it yourself, which is helpful if you are writing a lot of small, inline music examples.

To pin your examples to a different default release, specify `version`:

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      defaults: {
        version: "2.24.0"
      }
    }),
  ],
});
```

Blocks with an explicit `\version` declaration always use that version, regardless of the global config.

#### `defaults.resolution`

**Type:** `number`  
**Default:** `144`

Resolution to render `.png` files, in DPI. Only applies when `format` is set to `png`.

#### `defaults.crop`

**Type:** `boolean`  
**Default:** `false`

When `true`, the margins are removed and the page is cropped to fit the rendered staves. Leave as `false` to preserve full page dimensions.

#### `defaults.staffSize`

**Type:** `number`  
**Default:** `20`

Global staff height, in points. Typically somewhere between `11` (pocket scores) and `26` (large print). Smaller values will render thicker symbols and strokes to accommodate for their size. Keep this in mind if you are using responsive-width images.

For details on how to change the size of individual staves, refer to [setting the staff size](https://lilypond.org/doc/v2.26/Documentation/notation/setting-the-staff-size).

#### `defaults.paperSize`

**Type:** `string`  
**Default:** `"a4"`

Named paper size, e.g. `"letter"`, `"a3"`, `"tabloid"`. See LilyPond's [predefined paper sizes](https://lilypond.org/doc/v2.26/Documentation/notation/predefined-paper-sizes) for the full list.

### `format`

**Type:** `"svg" | "png"`  
**Default:** `"svg"`

The output format passed to the LilyPond binary. Controls how the rendered output is embedded in the page.

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      format: "png"
    }),
  ],
});
```

### `outputDir`

**Type:** `string`  
**Default:** `"_lilypond"`

Directory name that rendered scores are written to, relative to Astro's [`publicDir`](https://docs.astro.build/en/reference/configuration-reference/#publicdir) (`./public` by default).

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      // change output from `public/_lilypond` to `public/scores`
      outputDir: "scores"
    }),
  ],
});
```

:::note

Output files will be built, pruned, and rebuilt as needed. You can commit the generated files to cut down on build times; if the score hasn't changed, `astro-lilypond` will skip regenerating the image.

If you prefer to exclude generated images from your repository and build them from scratch each time, add the output directory to your `.gitignore`:

```ini title=".gitignore"
# exclude generated LilyPond images
public/_lilypond
```

:::
