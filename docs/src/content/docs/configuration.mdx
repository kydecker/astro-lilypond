Pass options to the integration when registering it in `astro.config.mjs`:

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import lilypond from 'astro-lilypond';

export default defineConfig({
  integrations: [
    lilypond({
      // config
    }),
  ],
});
```

## Options

### `version`

**Type:** `string`  
**Default:** `undefined`

All LilyPond content must specify `\version` at the start so that the compiler knows how to output the score.

<FencedLilypond code={versionExample} />

Specify `version` in the config to apply it to all LilyPond blocks by default:

```js
// astro.config.mjs  
export default defineConfig({
   integrations: [
    lilypond({
      version: "2.24.0"
    }),
  ],
});
```

With `version` set in the config, blocks can omit it.

Blocks with an explicit `\version` declaration will always use that version, regardless of the global config.

### `format`

**Type:** `"svg" | "png"`  
**Default:** `"svg"`

The output format passed to the LilyPond binary. Controls how the rendered output is embedded in the page.

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      format: "png"
    }),
  ],
});
```

### `resolution`

**Type:** `number`  
**Default:** `144`

Resolution to render `.png` files, in DPI. Only works when `format` is set to `png`.

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      format: "png",
      resolution: 300
    }),
  ],
});
```

### `crop`

**Type:** `boolean`  
**Default:** `true`

When `true`, the margins are removed and the page is cropped to fit the rendered staves.

Set to `false` to preserve full page dimensions:

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      crop: false
    }),
  ],
});
```

### `outputDir`

**Type:** `string`  
**Default:** `"_lilypond"`

Directory name that rendered scores are written to, relative to Astro's [`publicDir`](https://docs.astro.build/en/reference/configuration-reference/#publicdir) (`./public` by default). Files in this directory are served at `/` during dev and copied to your build directory during build.

```js
// astro.config.mjs
export default defineConfig({
  integrations: [
    lilypond({
      outputDir: "scores"
    }),
  ],
});
```

:::note
Output files will be built, pruned, and rebuilt as needed. If you prefer not to have built images from `astro-lilypond` clutter up your repository or show up in diffs, add the output directory to your `.gitignore`:

```.gitignore title=".gitignore"
# exclude generated LilyPond images
public/_lilypond
```

Alternatively, you can commit the generated files to cut down on build times. If the score hasn't changed, `astro-lilypond` will skip regenerating the image.

:::
